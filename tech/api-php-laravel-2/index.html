Laravel 中文文档

升级 Composer

1

$ composer self-update --2

新建项目

1
2
3

$ composer global require laravel/installer
$ laravel new "project name"
$ php artisan serve

将代码部署到服务器后需要 composer install 安装依赖。
注意 PHP 要提前安装 fileinfo 扩展，并启用 putenv 和 proc_open 两个函数。

然后 chmod -R 777 storage bootstrap/cache 打开日志、缓存、sessions 等等杂项的写入权限，就能在生产环境上跑了。

配置优雅链接

将 运行目录 设置为 /public，然后编辑 Nginx 的配置文件：

location / {
    try_files $uri $uri/ /index.php?$query_string;
}

拉取仓库配置见前文《API 重建计划 #1 Golenil》，此处不再赘述。

引入 Laravel Octane

虽然目前（2021.5.18）的 Laravel Octane 不尽如人意，但毕竟是提升，有总比没有好。

升级 PHP 8

Laravel Octane 需要 PHP 8。
升级过程参考 《Ubuntu 20.04 / 18.04 下如何安装或更新至 PHP 8？》。

更新软件源

1

$ sudo apt update && apt upgrade -y

添加 PPA

1
2
3

$ sudo apt install software-properties-common
$ sudo add-apt-repository ppa:ondrej/php
$ sudo apt update

安装 PHP 8

1
2

$ sudo apt install php8.0-fpm -y # Nginx
$ sudo apt install php8.0 -y     # Apache

安装常用扩展

1

$ sudo apt install php8.0-common php8.0-mysql php8.0-xml php8.0-curl php8.0-gd php8.0-imagick php8.0-cli php8.0-dev php8.0-imap php8.0-mbstring php8.0-opcache php8.0-soap php8.0-zip -y

$ composer require laravel/octane
